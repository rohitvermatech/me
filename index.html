About

The Short Version:
Engineer Who Builds Things That Work.

I started building chatbots in 2017. Not the kind that reply with canned responses. Real conversational systems that connect to APIs, pull live data, and handle multi-platform messaging across WhatsApp, Telegram, Facebook Messenger, and web. That was my entry point into engineering, and it taught me how systems actually talk to each other.

From there I kept stacking skills. Backend APIs. Full stack web development. SAP and NetSuite integrations. Power Automate Desktop for RPA. Multi-platform social media chatbot deployments. Every project pushed me to learn something new, and I did. Not from courses. From docs, from doing, from reading instructions and figuring it out hands-on.

That is still how I learn. Give me documentation and access, and I will deploy it. New language, new cloud platform, new API. I read the docs, follow the steps, test it, break it, fix it, and ship it. I have done this across enough stacks now that picking up new technology is just part of the process.

I use AI every single day. OpenAI, Gemini, Claude. Not as chat toys, but as real components inside real systems. I build agentic workflows, RAG pipelines, content automation, intelligent response agents. I also use AI to write code faster, what people call vibe coding. I understand the architecture, I know what I need to build, and I use AI to accelerate the execution. The result is production-quality systems shipped in a fraction of the time.

My core strengths: API integration (if it has an API, I can connect it), chatbot development across any platform, workflow automation, and learning fast enough to be dangerous with any new tool within a week.
